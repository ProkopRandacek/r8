# code style
I'm not really keen on these, just want to keep it consistent.

- all `.h` files use the ifndef-define-code-endif trick
- all `.h` files include `common.h` as the first include
- all `.h` files start with `//vim: ft=c`
- do use the `LIKE_MALLOC`, `PURE` and `CONST` attributes
- do use the restrict keyword
